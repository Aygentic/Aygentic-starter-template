# CLAUDE.md

## Project Overview

Microservice starter template: FastAPI backend + React frontend. Clerk JWT auth (received from external dashboard), Supabase REST, Entity CRUD reference pattern. Auto-generated API client. Docker Compose monorepo with Traefik reverse proxy.

## Technology Stack

| Category | Technology |
|----------|-----------|
| Backend | Python >=3.12, FastAPI >=0.114.2 |
| Auth | Clerk JWT (received from external dashboard, no login flow) |
| Database | Supabase REST |
| Migrations | Supabase CLI (`supabase/migrations/`) |
| Observability | structlog >=24.1.0 (JSON/console), Sentry (non-local) |
| Resilience | httpx + tenacity (retry, circuit breaker) |
| Frontend | TypeScript 5.9, React 19.1, Vite 7.3 (SWC) |
| Routing | TanStack Router 1.157+ (file-based) |
| Server State | TanStack Query 5.90+ |
| Styling | Tailwind CSS 4.2, shadcn/ui (new-york) |

**Key Libraries:** Zod 4.x, React Hook Form 7.x, Axios 1.13, lucide-react, sonner, clerk-backend-api, supabase-py

## Architecture Overview

**Pattern:** Layered monorepo — `backend/` (Python) + `frontend/` (TypeScript)

```
backend/app/
  api/routes/        # FastAPI endpoint handlers (thin → delegate to services)
  core/              # auth.py, config.py, errors.py, http_client.py,
                     # logging.py, middleware.py, supabase.py
  models/            # Pydantic package: common.py, auth.py, entity.py
  services/          # Business logic: entity_service.py
frontend/src/
  routes/            # TanStack file-based routing (/ and /entities)
  components/        # UI (Entities/, Common/, Sidebar/, Pending/, ui/)
  client/            # Auto-generated OpenAPI client — DON'T EDIT
  hooks/             # useAuth (token-based), useCustomToast, useCopyToClipboard
```

**Key Patterns:**
- Backend: route → `entity_service.py` → Supabase REST (no ORM)
- Auth: Backend uses `PrincipalDep` (Clerk JWT → `Principal`). Frontend receives JWT from external dashboard via `?token=` URL param or `VITE_DEV_TOKEN`, stores in `localStorage`
- Errors: `ServiceError(status_code, message, code)` — 3-arg constructor
- DB: `SupabaseDep` from `app.core.supabase`
- `src/client/`, `src/routeTree.gen.ts`, `src/components/ui/` are auto-generated — don't edit

**Key Files:** `backend/app/main.py` (entry + lifespan), `backend/app/core/config.py` (settings), `frontend/src/main.tsx` (React + QueryClient + Router), `frontend/src/hooks/useAuth.ts` (token management)

## Development Commands

```bash
# Setup
cd backend && uv sync                # Backend deps
bun install                           # Frontend deps
cd backend && uv run prek install -f  # Pre-commit hooks
docker compose watch                  # Start full stack

# Development
docker compose watch                  # Full stack (recommended)
bun run dev                           # Frontend only (:5173)
cd backend && fastapi dev app/main.py # Backend only (:8000)

# Testing
bash ./scripts/test.sh                # All backend tests (Pytest)
bash ./scripts/test-local.sh          # Backend tests locally
bunx playwright test                  # Frontend E2E
uv run pytest backend/tests/unit/ -v  # Unit tests only

# Quality
bun run lint                          # Frontend (Biome)
uv run ruff check --fix               # Backend lint
uv run ruff format                    # Backend format
uv run mypy backend/app               # Backend type check

# Utilities
bash ./scripts/generate-client.sh     # Regenerate OpenAPI client
supabase db push                      # Apply Supabase migrations
```

## Code Conventions

- **Backend:** snake_case files/functions, PascalCase classes, absolute imports `from app.*`, Ruff + strict mypy
- **Frontend:** PascalCase components, camelCase utils, `@/` alias → `./src/*`, Biome (double quotes, no semicolons)

## Testing

**Backend:** Pytest in `backend/tests/` — `uv run pytest backend/tests/path/to/test.py`
- `tests/unit/` — Service layer, models, core modules (MagicMock Supabase, no DB needed)
- `tests/integration/` — Route handlers with TestClient + dependency overrides

**Frontend:** Playwright E2E in `frontend/tests/` — `bunx playwright test tests/file.spec.ts`
- `tests/entities.spec.ts` — Entity CRUD lifecycle (create, edit, delete)
- `tests/navigation.spec.ts` — Sidebar links, theme toggle, user menu
- Auth: token injection via `page.addInitScript()` (no login form)

**Coverage target:** 90% backend (enforced in CI). See `@docs/testing/strategy.md`.

## Database & Migrations

**Supabase CLI:** `supabase migration new <name>` → `supabase db push`

Always review autogenerated migrations before applying. See `@docs/data/models.md`.

## Known Issues

- `SUPABASE_SERVICE_KEY` and `CLERK_SECRET_KEY` validated on startup — errors in production if weak/missing
- Backend syntax errors crash dev container — restart with `docker compose watch`
- Pre-commit hook auto-regenerates frontend SDK on backend changes
- `ServiceError` uses 3-arg form `(status_code, message, code)` — not 4
- `gh` CLI may default to upstream repo — use `gh repo set-default` if needed
